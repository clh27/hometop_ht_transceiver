Tabelle 2: Telegramm-Details
ht_transceiver Steuer-Telegramme





















Datum: 30.10.2014










Version: 0.0.2























Telegramme Bedeutung Direction Details B0 B1 B2 B3 B4 D-Byte 0 | CRC D-Byte 1 D-Byte x CRC | Daten
1.1 Daten senden auf Bus → B3 steht z. Zeit fest auf Wert: (11)hex und ist für: Anzahl maximale- und aktuelles- Paket vorgesehen. Das obere Nibbel ist: maximale Anzahl Pakete, das untere Nibbel ist: das aktuelle Paket. Beispiel: (21)hex → Maximal 2 Pakete und aktuell ist dies das 1. Paket. Zur Zeit wird dieser Wert nicht ausgewertet. Jedes Paket wird mit Antwort 1.2 quittiert. Maximal 55 Daten-Bytes erlaubt. # ! S packets length Daten B0 Daten B1 Daten Bx CRC-Byte



B4 gibt die Anzahl der Daten-Bytes (ohne CRC-Byte) an.








1.2 Antwort auf Senden ← B3 ist 0, wenn kein Fehler vorhanden ist. # ! s 0 0 CRC-Byte - - -


← B3 ist negativ (Wert > 128), wenn ein Fehler aufgetreten ist. Der Wert bestimmt die Fehlerart. # ! s error 0 CRC-Byte - - -


← B3 ist positiv (Wert < 57), wenn das Telegramm gesendet werden konnte. Es gibt dabei die Anzahl der gesendeten Daten-Bytes + CRC an. # ! s length 0 CRC-Byte - - -


























2.1 Mode einstellen → B3 gibt den zu setzenden Arbeits-Mode an. Diese Einstellung wird durch einen HW-Reset auf die Konfigurationswerte zurückgesetzt. Dieses Kommando wird mit Telegramm: 2.2 quittiert. (Hinweis: Kommandos an den ht_transceiver müssen immer mit Header gesendet werden.) # ! M mode 0 CRC-Byte - - -



Werte für Mode (B3):










→ D0:=0 → OFF, keine Busdaten-Übertragung an Master, kein senden von Busdaten auf dem HT-Bus. # ! M 0 0 CRC-Byte




→ D0:=1 → HEAD, Busdaten-Übertragung an Master (mit Header), kein Senden von Busdaten auf dem HT-Bus. # ! M 1 0 CRC-Byte




→ = (F0)h → RESET, Reset des HT-transceivers. Telegramm wird nicht quittiert. # ! M (F0)h 0 CRC-Byte















2.2 Antwort auf: Mode einstellen ← B3 ist 0, wenn kein Fehler vorhanden ist. # ! m 0 0 CRC-Byte - - -


← B3 ist negativ (Wert > 128), wenn ein Fehler aufgetreten ist. Der Wert bestimmt die Fehlerart. # ! m error 0 CRC-Byte - - -


























3.1 Konfiguration einstellen → B3 bestimmt welcher Konfigurations-Anteile eingestellt werden sollen. Die Werte werden im EEProm abgelegt und sind erst nach einem HW-Reset aktiv. # ! C Konfig length Daten Byte 0 …




Werte für Konfig (B3):










→ B3:=1 → System-Startmode. D0 ist der Konfigurations-Wert.










→ D0:=0 → OFF, keine Busdaten-Übertragung an Master, kein senden von Busdaten auf dem HT-Bus. # ! C 1 1 0 CRC-Byte



→ D0:=1 → HEAD, Busdaten-Übertragung an Master (mit Header), kein Senden von Busdaten auf dem HT-Bus. # ! C 1 1 1 CRC-Byte



→ D0:=2 → TX, Senden von Busdaten auf dem HT-Bus, Busdaten-Übertragung an Master im RAW-Format (ohne Header). # ! C 1 1 2 CRC-Byte



→ D0:=3 → HEADTX, Busdaten-Übertragung an Master (mit Header), Senden von Busdaten auf dem HT-Bus. # ! C 1 1 3 CRC-Byte

















B3:=2 → Transceiver Bus-Geräteadresse. D0 ist die Geräteadresse. # ! C 2 1 xy CRC-Byte




Unzulässige Werte werden nicht übernommen und mit Fehler quittiert. Werte hängen vom Bus-System ab. !!!!!!! Achtung !!!!!!! Änderung der Default-Adresse kann zu Problemen am HT-Bus führen.











Heatronic Wertebereich: (0a…7F)hex. Default-Wert ist (0d)hex.





















3.2 Antwort auf Konfigurationseinstellung ← B3 ist 0, wenn kein Fehler vorhanden ist. # ! c 0 0 CRC-Byte - - -


← B3 ist negativ (Wert > 128), wenn ein Fehler aufgetreten ist. Der Wert bestimmt die Fehlerart. # ! c error 0 CRC-Byte - - -


























4.1 Konfigurationswerte abfragen → B3 bestimmt welcher Konfigurations-Anteil abgefragt werden soll. # ? C Konfig 0 CRC-Byte


4.2 Antwort auf: Konfigurations-Werte abfragen ← D0 gibt den aktuell eingestellten Konfigurations-Wert an, der zu der in B3 angegebenen Konfiguration gehört. Ungültige Konfigurations-Werte werden mit (0xff)h zurückgegeben. # ? c Konfig 1 0 … 3 bzw. xy CRC-Byte



























5.1 Bus-Telegramm Ausgabe für Heatronic Telegramme (Junkers) ← B3 steht z. Zeit fest auf Wert: (11)hex und ist für: Anzahl maximale- und aktuelles- Paket vorgesehen. Das obere Nibbel ist: maximale Anzahl Pakete, das untere Nibbel ist: das aktuelle Paket. Beispiel: (21)hex → Maximal 2 Pakete und aktuell ist dies das 1. Paket. Zur Zeit wird dieser Wert nicht ausgewertet. Eine Quittierung ist nicht erforderlich. Maximal 55 Daten-Bytes erlaubt. # H R packets length Daten Byte 0 Daten Byte 1 Daten Byte x CRC-Byte
5.2 Antwort entfällt

Errorcode Bedeutung Kürzel (define)
